Что в модели:

Участники (агенты) с весом (stake) и надежностью (вероятность быть онлайн).
Группировка по сущностям (entityId), чтобы индекс Накамото был устойчив к Sybil-сплитам.
Консенсус "кворум": в каждом раунде выбираются подписанты, пока не покрыт порог голосов Q (например 51% или 67%).
Параметризуем политику отбора подписантов: greedy (самые тяжелые), weighted (взвешенно по стейку без возвращения), uniform (равновероятно).
Параметризуем схему наград (равномерно между подписантами по умолчанию).
Измеряем:
Индекс Накамото (по сущностям, порог 50%).
Долю топ-1 сущности.
Gini (опционально).
Метаметрика «давление к централизации»: наклон линейной регрессии индекса Накамото во времени.

Как запускать:

Скрипт index.ts запускает несколько примерных сценариев и пишет результаты в out/results.json и out/results.csv.
Есть простой HTML с Plotly для визуализации N(t) по сценариям.
